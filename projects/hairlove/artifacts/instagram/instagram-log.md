## 2026-05-16 | КОРНЕВАЯ ПРИЧИНА найдена (конец хождения по кругу)

Проблема повторяется неделями потому, что каждый раз чинили НЕ ТОТ СЛОЙ (включая идею «12 попыток» — это была заплатка на симптом).

**Корень: GitHub Actions cron — неправильный инструмент для постинга в точное время.**
- Scheduled-воркфлоу на бесплатных раннерах = общая очередь без гарантий времени. GitHub официально: запуск может задержаться на часы или быть пропущен.
- «Окно попыток» → пост уходит в случайный момент. Пост #5: 20:55 UTC = 23:55 Киев вместо 20:00.
- Правка крона/окна это НЕ лечит — проблема в платформе планировщика, а не параметрах.

**Решение (ожидает «да» Павла): перенести планировщик GitHub → VPS.**
- VPS 188.166.67.237 уже работает (assistant, systemd, instagram_client.py, токены).
- systemd-timer `OnCalendar=20:00 Europe/Kiev` → срабатывает в 20:00 ± секунды, детерминированно.
- GitHub Actions убрать из пути таймингов (оставить как ручной запуск).
- Настройка разовая через существующий SSH-деплой.
