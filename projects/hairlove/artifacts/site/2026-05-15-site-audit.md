### B2B форма — что реально происходит с заявками
**КРИТИЧЕСКАЯ ПРОБЛЕМА:** форма НЕ отправляет данные через Formspree.

Несмотря на то что Formspree ID `meenvegv` зафиксирован в team-state.md как "подключён", в коде форма работает через `mailto:`:

```javascript
const mailto = `mailto:info@hairlove.com?subject=...&body=...`;
window.location.href = mailto;
```

**Что происходит по факту:** при нажатии "Надіслати заявку" открывается почтовый клиент пользователя (не у всех он настроен). Email `info@hairlove.com` — скорее всего не существующий адрес. Formspree (`action="https://formspree.io/f/meenvegv"`) в HTML форме НЕ подключён.

**Итог:** заявки с сайта НЕ доходят до Павла/Максима.

## 4. СЛЕДУЮЩИЙ ШАГ — 2 улучшения с наибольшей отдачей

### Приоритет 1: ПОЧИНИТЬ ФОРМУ (1-2 часа)
Заменить `mailto:` на реальный Formspree POST-запрос:

```html
<form action="https://formspree.io/f/meenvegv" method="POST" id="contact-form">
```

И в JS-обработчике убрать `window.location.href = mailto` — заменить на `fetch` с Formspree API. Данные начнут приходить на email Павла сразу.

### Приоритет 2: SILK teaser-карточка (30 минут)
В секцию `#products` добавить 4-ю карточку SILK 30 мл с бейджем "Скоро в продажі". Состав: шёлк, инфузия. Показывает что линейка растёт — работает на доверие дистрибутора.
